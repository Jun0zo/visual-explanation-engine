OAuth 로그인 흐름

사용자, 브라우저, 앱 서버, 인증 제공자가 코드를 주고받아 로그인하는 구조입니다.

등장인물

사용자

로그인 버튼을 누릅니다.

브라우저

리다이렉트를 따라갑니다.

앱 서버

세션을 만듭니다.

Provider

코드와 토큰을 발급합니다.

1

로그인 시작

사용자가 앱에서 로그인 버튼을 누르면 인증 서버로 이동합니다.

2

동의 화면

제공자가 사용자에게 권한 동의를 받습니다.

3

코드 반환

브라우저는 authorization code를 앱 callback으로 가져옵니다.

4

토큰 교환

앱 서버가 코드를 access token으로 바꾸고 세션을 만듭니다.

중요한 전제

authorization code는 access token이 아닙니다. 서버가 provider와 직접 교환해야 합니다.

code ≠ token

보안 포인트

state는 CSRF를 막기 위한 값이고, redirect URI는 미리 등록되어 있어야 합니다.

state 검증

흔한 오해

브라우저가 토큰을 직접 들고 오는 흐름으로 이해하면 서버 세션과 code exchange를 놓칩니다.
