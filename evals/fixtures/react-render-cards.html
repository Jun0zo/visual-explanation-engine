React 렌더링을 이해하기

상태 업데이트, 렌더, 커밋, 이펙트가 어떤 순서로 일어나는지 단계별 카드로 정리했습니다.

렌더링 흐름 핵심 개념

1

이벤트

사용자가 버튼을 누르면 이벤트 핸들러가 실행되고 setState가 예약됩니다.

2

렌더

React는 컴포넌트 함수를 다시 호출해서 다음 UI 모양을 계산합니다.

3

커밋

계산된 차이를 실제 DOM에 반영합니다. 화면 변경은 이 시점에 보입니다.

4

이펙트

화면이 반영된 뒤 useEffect가 실행됩니다. 여기서 외부 동기화를 합니다.

function Counter() {
  const [count, setCount] = useState(0);
  useEffect(() => log(count), [count]);
  return <button onClick={() => setCount(count + 1)}>
    {count}
  </button>;
}

중요한 점

setState는 값을 바로 바꾸는 명령이 아니라 다음 렌더를 예약하는 신호입니다.

흔한 오해

이벤트 함수 안에서 DOM이 즉시 바뀐다고 생각하면 effect 순서를 잘못 읽게 됩니다.
