React 렌더는 코드가 다시 지나간 흔적이다

버튼 클릭은 값을 즉시 밀어 넣는 일이 아니라, 같은 컴포넌트 함수를 새 상태로 다시 통과시키는 예약입니다.
1234567function Counter() {
  const [count, setCount] = useState(0);
  useEffect(() => log(count), [count]);
  return <button
    onClick={() => setCount(count + 1)}
  >{count}</button>;
}
setState는 예약 신호 이 줄이 다음 render를 큐에 넣습니다.
화면은 commit에서 바뀜 버튼 숫자는 여기서 0에서 1로 보입니다.
effect는 뒤따라 옴 그래서 DOM 동기화와 로그가 여기로 갑니다.
